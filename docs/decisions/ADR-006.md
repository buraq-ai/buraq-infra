# ADR-006: Vector Database Selection — ChromaDB + Milvus

## Status
Accepted

## Context
The RAG pipeline requires a Vector Database to store and
retrieve document embeddings. Three candidates were evaluated:
ChromaDB, Milvus, and Pinecone.

## Decision
We will use ChromaDB for development and testing.
We will migrate to Milvus for production deployment.
Pinecone is eliminated due to data sovereignty concerns.

## Consequences
- ChromaDB runs fully locally — no data leaves the enterprise
- Native LangChain integration available for both options
- Milvus migration must be planned before production release
- Spike confirmed ChromaDB works correctly with similarity search

# ADR-002: JWT-Based Stateless Authentication

## Status
Accepted

## Context
We need an authentication mechanism that works across
two separate microservices (Java backend + Python AI service)
without sharing session state.

## Decision
We will use Spring Security with JWT tokens.
The Java backend issues tokens. The Python service
validates them independently using the shared secret.

## Consequences
- Tokens must be short-lived (15–30 min)
- A refresh token strategy must be implemented
- Token secret must be stored securely (environment variable, never hardcoded)

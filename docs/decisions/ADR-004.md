# ADR-004: PostgreSQL for Relational Data

## Status
Accepted

## Context
The system needs a reliable relational database to store
users, tickets, roles, and audit logs.

## Decision
We will use PostgreSQL as the primary relational database.
It will be managed by the Java Spring Boot backend.

## Consequences
- PostgreSQL must be containerized using Docker
- Database migrations will be managed using Flyway
- Backups must be scheduled and tested regularly
- Connection pooling must be configured for performance

# ADR-003: LLM Strategy — OpenAI Primary + Ollama Fallback

## Status
Accepted

## Context
The system needs a reliable LLM for answering employee questions.
We need to balance answer quality, cost, and Moroccan data
sovereignty requirements.

## Decision
We will use OpenAI API as the primary LLM provider.
Ollama running locally will serve as a fallback to ensure
data never leaves the country when required.

## Consequences
- OpenAI API costs must be monitored and capped
- Ollama requires local infrastructure setup
- Switching logic must be implemented in the AI microservice
- Answer quality may differ between providers

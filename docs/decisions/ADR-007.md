# ADR-007: Notification Delivery Method — In-App Primary + SMTP Email Backup

## Status
Accepted

## Context
The Buraq-AI platform needs to notify users about ticket
events (created, updated, assigned). Users may not always
be logged in when events occur.

## Decision
We will use a dual delivery approach:
1. In-app notifications stored in PostgreSQL as the primary
   method — polled by Angular every 30 seconds, marked as
   read on click.
2. SMTP email via Spring Boot JavaMailSender as the secondary
   backup method — sent in addition to the in-app notification,
   with failures never breaking the main flow.

SMTP credentials are stored as environment variables
(MAIL_USERNAME, MAIL_PASSWORD) — never hardcoded.

## Consequences
- In-app notifications require Angular polling (browser
  must be open to receive them)
- SMTP configuration adds external dependency on email
  provider (Gmail SMTP or similar)
- Email delivery not guaranteed due to spam filters
- Thymeleaf templates needed for HTML email formatting
- app.mail.enabled feature flag allows disabling email
  in development environments
- Email failures are caught and logged — they never
  break the ticket flow
